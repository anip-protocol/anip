ANIP GTM Showcase

Local Runbook

Commands and structure for running the GTM showcase locally.

Architecture

  • Postgres stores the Maven CRM dataset.
  • dbt builds the GTM analytics models.
  • Metabase verifies the modeled BI slices.
  • Four generated ANIP services expose pipeline, enrichment, prioritization, and outreach capabilities.
  • The agent UI consumes ANIP discovery and selects the target service automatically.

Run Stack

export OPENAI_API_KEY=...
export ANIP_AGENT_MODEL=gpt-5.4-mini
docker compose -p anip-gtm-local -f examples/showcase/gtm/docker-compose.language-parity-python.yml up -d --build

Run 350 + 140 Question Gate

python3 examples/showcase/gtm/scripts/generated_stack/run_question_bank.py \
  --runtime-url http://127.0.0.1:4310 \
  --all

OUT=output/gtm-variation-question-runs
mkdir -p "$OUT"
for phase in 1 2 3 4 5 6 7; do
  python3 examples/showcase/gtm/scripts/generated_stack/run_phase1_regression.py \
    --runtime-url http://127.0.0.1:4310 \
    --cases "docs/examples/gtm-showcase/variation-question-banks-v3/phase${phase}-variation-bank-20.json" \
    --output-dir "$OUT"
done

Custom Bundles

Generated services are built from the GTM ANIP contract plus reviewed custom-code bundles under examples/showcase/gtm/custom-code-bundles/. Replace implementation seams there without changing the public contract shape.
