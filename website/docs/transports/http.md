# HTTP

HTTP is the default ANIP binding.

It is the easiest place to start because:

- it is broadly understood
- it works well with browsers and Studio
- it aligns naturally with service deployment
- it is implemented across all current runtime families

## What HTTP covers

Over HTTP, ANIP exposes:

- discovery
- manifest
- JWKS
- token issuance
- permission discovery
- invoke
- audit query
- checkpoints

## Why it matters

HTTP is the main path for:

- public docs and examples
- showcase apps
- Studio inspection and invocation
- REST and GraphQL adapter composition
