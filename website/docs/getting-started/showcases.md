# Start With Showcases

Showcases are examples and proofs. They are not the Studio authoring product itself.

Use this page to choose where to start. Use the full [Showcases reference](/docs/showcases/overview) for commands, package details, local stacks, and deeper walkthroughs.

## Pick a showcase

| If you want to see | Start with |
| --- | --- |
| A full multi-service ANIP agent application generated in five languages | [GTM Agent](/docs/showcases/overview#gtm-agent-showcase) |
| One signed contract generated across Python, TypeScript, Go, Java, and C# | [GTM Agent Showcase](/docs/showcases/gtm-agent/overview) and [GTM Language Parity](/docs/showcases/gtm-language-parity) |

| Small protocol examples for learning primitives | [Introductory examples](/docs/showcases/overview#introductory-domain-showcases) |

## What to inspect

For any showcase, inspect:

- The Studio project or source starter.
- The signed Registry package.
- The generated service definition.
- The capability list and input-resolution metadata.
- Approval, denial, restriction, clarification, recovery, and audit behavior.
- Generated service code and custom bundle seams.
- Live smoke scripts or scenario/question-bank evidence.

The point of a showcase is to teach where behavior belongs: in the contract and service enforcement, not in prompts or raw tool descriptions.
